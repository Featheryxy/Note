## 重构原则



何为重构:对软件内部结构的一种调整，目的是在不改变软件可观察行为的前提下，提高其可理解性，降低其修改成本

两顶帽子: 添加新功能和重构。添加新功能时，我不应该修改既有代码，只管添加新功能;重构时我就不能再添加功能，只管调整代码的结构

何时重构:Don Roberts给了我一条准则：第一次做某件事时只管去做；第二次做类似的事会产生反感，但无
论如何还是可以去做；第三次再做类似的事，你就应该重构。
