### 编码原则

1. 奥卡姆剃刀
2. 就近原则,
2. 细化粒度，（推迟对象的创建，...)
4. 全集实现（书写判断时，先找到全集，保证流程的封闭性(尤其是在做连接查询时，注意悬浮元组是否为空的判断)
5. 先利用数据本身来判断，实在不行再加标志位
6. 合并代码

### 关联关系

任何两个对象之间在数量上都存在关系

1. 一对一
2. 一对多
3. 多对多

> 一为高层, 多为低层，越高层越抽象

```
1个学生 对应 1个班级
1个班级 对应 多个学生


班级:学生 = 1:n

把1的主键放着n的表中

班级表（class_id, nums）
学生表（stu_id, nmae, class_id）
```

修改功能和扩展功能使用extends 

```java
class A{
	public void errorMethod(){
	
	}	
}

// 修改功能 
class B() extends A{
	@Override
	public void errorMethod(){
		// 覆盖原有功能
	}	
}

// 扩展功能
class B() extends A{
	@Override
	public void errorMethod(){
		super.errorMethod();
        // 新功能
	}	
}
```
